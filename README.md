# WHO’S NEXT?

発表会などで **発表順をランダムに決めるための Web アプリ** です。
一度選ばれた人は二度と出ず、進行状況は自動保存されます。

サーバ不要・HTML/CSS/JavaScriptのみで動作します。

---

## 特徴

* 発表者を **ランダムかつ重複なし** で抽選
* 一度に **複数人（PICK数）** を選択可能
* 抽選結果・進行状況を **自動保存**
* ページを閉じても、再読み込みしても **前回の状態を復元**
* **リセット操作** 以外では初期化されない
* PC / スマートフォン対応

---

## 使い方

### 1. 起動

`index.html` をブラウザで開いてください。
（ローカルでOK。サーバは不要です）

---

### 2. 発表人数（PICK）を設定

画面左下の **PICK** に、次に選びたい人数を入力します。

例：

* `1` → 1人ずつ発表順表示
* `5` → 5人まとめて発表順表示

※ 残り人数を超える数を入れても、自動で調整されます。

---

### 3. 抽選する

中央の **START** ボタンをクリック（またはタップ）します。

* ランダムに選ばれた名前が表示されます
* この時点で **抽選は確定** します（やり直し不可）

---

### 4. 次に進む

表示されている **名前をクリック** すると、START画面に戻ります。

* すでに出た人は、次回以降選ばれません。
* 右側（スマホでは下側）の **history** に、出た順で記録されます

---

### 5. 全員終了したら

全員が選ばれると、STARTボタンは無効になります。
再度使う場合は **リセット** を行ってください。

---

## リセット方法

![resetbutton](assets/resetbutton_screenshot.png)
画面左上の **正方形が重なったのマーク** がリセットボタンです。

* タップすると **確認ダイアログ** が表示されます
* OK を押すと、進行状況がすべて初期化されます

* 一部のブラウザ（モバイル版Braveなど）では
確認ダイアログがブロックされ、リセット機能が使用できません。
スマホでは **Chrome / Safari** の使用を推奨します。

---

## HISTORY（履歴）について

* 出た順に 01, 02, 03… と番号付きで表示されます
* まだ出ていない人は空欄表示になります

---

## 対応ブラウザ

動作確認済み：

* Brave（macOS）
* Chrome（iOS / macOS）
* Safari（iOS / macOS）

注意：

* **Brave（モバイル版）** は
  セキュリティ機能により確認ダイアログが表示されず、リセット機能が使用できません

---

## カスタマイズ

### 名簿を変更したい場合

`script.js` 内の `ROSTER` を編集してください。

```js
const ROSTER = [
  { id: '01', name: '山田 太郎' },
  { id: '02', name: '佐藤 花子' },
  ...
];
```

* `id` は内部管理用（連番でOK）
* `name` が画面に表示されます
